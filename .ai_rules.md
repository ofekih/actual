# AI Behavioral Rules

1. NEVER assume approval to begin making code changes.
2. A question like 'what is next?' or 'how does this work?' is purely informational and NEVER implies a green light to execute the next steps of a plan.
3. Always wait for EXPLICIT approval (e.g., 'go ahead', 'approved', 'do it') before modifying any source code or triggering irreversible actions.
4. Prevent noisy diffs: Use small, highly targeted search-and-replace blocks that strictly match only the line being altered. Avoid replacing large chunks of code that might adjust spacing.
5. Do NOT run global auto-formatters (e.g. `yarn lint:fix`) on existing files unless explicitly requested or absolutely necessary to fix a critical style failure, as this reformats the whole file and obscures the actual changes.
